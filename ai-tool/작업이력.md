# AI 활용 페이지 작업 이력

최종 갱신: 2026-07-14 (서브탭 위치·메뉴명 변경, DEMO 레이아웃 통일, 줄바꿈/에러 표시 개선)

## 배경

OneWayTicketStudio 재직 중 Claude Code로 작업했던 내용(`클루드/` 폴더의 세션 로그·스킬·MCP 서버 등)을 분석하고,
그 내용을 바탕으로 포트폴리오의 **AI 활용 페이지**(`ai-tool/index.html`)를 기획·제작했다.

## 진행 순서

1. **클루드 폴더 분석**
   - `tmw-claude-migration-2026-05-29/` 안의 세션 로그 52개, 스킬 정의 31개, MCP 서버 2종(`tmw-analyzer`, `confluence-mcp-server`) 확인
   - 분석에 쓰인 스킬은 `session-insights` (세션 로그를 스스로 분석해 반복 패턴·스킬 후보를 뽑는 메타 스킬)
   - 결과: 타임라인, 모듈별 발전 과정, Claude 활용 패턴, 트러블슈팅 하이라이트(로그 필드 불일치로 인한 경제 지표 과소 집계 등) 정리

2. **AI 활용 페이지 기획**
   - 톤 결정: 기획 역량보다 **AI 활용 역량 자체를 주인공**으로
   - 데모 후보 중 **MapGenie**(맵 시각화, 메인) + **DataViewer**(데이터 조회, 보조) 선정
   - 계획을 `포트폴리오_제작_계획안.md`에 반영 (데모 데이터 처리 방침, 작업 순서 포함)

3. **MapGenie 데모 구현**
   - 실제 레벨 디자인 블루프린트 이미지(`LV_BCF_A/B.png` → 리사이즈 후 `assets/map/*.jpg`) 사용, 회사 대표 최종 컨펌 예정으로 명시
   - 가상 스폰 마커 데이터 작성, 층 전환·타입 필터·검색·팬/줌·상세 패널 인터랙션 구현
   - 피드백 반영: 마커가 실제 플레이 영역(벽 안쪽) 밖으로 나가지 않도록 좌표 재배치, 마커 수 약 3배 증가, 필터 목록 아이콘을 맵 위 마커와 동일한 SVG 디자인으로 통일

4. **DataViewer 데모 구현 및 재설계**
   - 1차: 단일 가상 테이블(`DT_SampleItem`)로 검색/정렬/필터
   - 2차: 요청에 따라 **Item → BaseItem → String 참조 체인** 구조로 전면 재설계 + **전역 키 검색**(테이블 3개 전체에서 키 값 검색) 핵심 기능 추가
     - 끊긴 참조(BaseItem 없음), 누락된 문자열(String 없음), 미사용 고아 데이터 3가지 시나리오를 실제로 검색해서 찾아낼 수 있도록 데이터에 의도적으로 심어둠
   - 테이블명에서 `DT_` 접두사 제거
   - 키 값(Index/BaseID/StringTableID/StringID)을 숫자 → 문자열 타입으로 전환
   - 인덱스 규칙을 원본 데이터 규칙에 맞춰 재구성: `Item.Index = BaseID + 등급순번(1~6, Junk~Legendary)`, String 키는 `ItemName_{BaseID}` 형식으로 변경
   - Item/BaseItem에 Desc(설명) 필드 추가, String 테이블에 English·Japanese 다국어 컬럼 추가
   - UX 개선: 검색 결과 요약 줄을 없애고 **테이블 탭 버튼 자체에 매치 건수 배지**로 통합, 테이블 목록도 전체 표시 대신 **검색 결과만 필터링**해서 표시

5. **작업 이력 관리 방식 정착**
   - 이 문서(`작업이력.md`)를 md로 정리해달라는 요청을 계기로, "같은 주제는 항상 같은 파일에 이어서 기록"하는 전역 스킬 `history-log`(`~/.claude/skills/history-log/SKILL.md`)를 신설
   - 날짜별로 새 파일을 만드는 기존 `session-log` 스킬과 역할을 분리 — 이 파일처럼 기능/페이지 단위로 누적되는 문서는 앞으로 `history-log`가 관리
   - 이번 갱신이 그 스킬로 남긴 첫 기록

6. **MapGenie 드롭 확률/아이템 검색 기능 추가**
   - 요청 배경: 실제(사내) 맵지니에 있던 "아이템 드랍률·드랍 아이템 표시", "아이템 이름 검색 시 해당 아이템을 드랍하는 오브젝트만 표시" 기능이 데모에는 빠져 있었음
   - `assets/map-drop-data.js` 신설: 가상 아이템 카탈로그 18종(등급 Junk~Legendary) + 드롭테이블 6종(등급별 확률·구성 아이템), ItemBox/Item 마커에 드롭테이블을 순환 배정
   - 상자·바닥 낙하물 마커 클릭 시 상세 패널에 **드롭 아이템 섹션** 추가 — 등급별 확률 배지 + 드롭 아이템 목록(등급 색상·수량)
   - 사이드 패널에 **"아이템 드롭 검색"** 입력창 신설 — 아이템명 검색 시 해당 아이템을 드롭하는 오브젝트만 지도에 표시(그 외는 숨김), 다른 층에 매칭 결과가 있으면 층 탭에 배지, "전체 초기화" 버튼으로 함께 리셋
   - 검증: 실제 실사(전 회사) 맵지니 코드(`클루드/.../MapGenie.js`)의 드롭 렌더링 로직(등급별 확률 배지, 아이템 그룹핑, 검색 매칭)을 참고해 데모 로직 설계 → Edge 헤드리스 + DevTools Protocol로 클릭/검색 상호작용을 스크립트로 구동해 동작 확인(상자 클릭 시 드롭 섹션 렌더링, "손전등" 검색 시 마커 56→18개로 필터링 등)

7. **MapGenie 레이아웃 버그 수정**
   - 증상: 드롭 아이템 목록이 길어지면서 우측 패널(`mg-side`) 콘텐츠가 지도(`mg-viewport`)보다 커지자, flexbox 기본값(`align-items: stretch`)에 의해 지도 박스까지 함께 늘어나 `aspect-ratio: 1/1`이 무시되고 정사각형 지도가 세로로 찌그러짐
   - 수정: `mg-demo-body`에 `align-items: flex-start` 적용해 지도가 항상 자기 너비 기준 정사각형을 유지하도록 하고, `mg-side`는 JS(`syncSideHeight`)로 지도 높이에 맞춰 인라인 `height`를 동기화 + `overflow-y: auto`로 내부 스크롤 처리 → 콘텐츠가 길어져도 우측 패널만 스크롤되고 지도 크기는 항상 고정
   - 창 리사이즈 시 재동기화, 모바일 스택 레이아웃(760px 이하)에서는 높이 고정을 자동 해제해 기존처럼 세로로 자연스럽게 쌓이도록 분기 처리
   - 검증: 헤드리스 브라우저로 클릭 전후 지도가 항상 640×640 정사각형을 유지하는지, 우측 패널만 `scrollHeight`가 늘어나 내부 스크롤(`scrollTop` 반영)되는지, 모바일 폭에서 세로 스택 + 높이 고정 해제가 정상 동작하는지 확인

8. **서브탭 내비게이션 위치 조정**
   - "AI 툴 활용 / 기획 및 프로토타입" 서브탭이 히어로(`content-header`) 아래에 있어 "AI TOOL" 섹션 하위 메뉴처럼 보인다는 피드백
   - `ai-tool/index.html`, `prototypes/index.html` 둘 다 서브탭을 상단 사이트 내비게이션 바로 아래로 이동
   - `assets/css/style.css`의 `.subtab-nav` 마진을 첫 자식 기준(`24px 0 0`)으로 재조정, `.subtab-nav + .content-header`에 `padding-top: 24px` 규칙을 추가해 간격 정리

9. **상단 메뉴명 변경: "AI 활용" → "프로젝트"**
   - 페이지가 이미 AI 툴 활용 + 기획/프로토타입 두 서브탭을 포함해 "AI 활용"이 내용을 다 못 담고, 사이트 전체가 "포트폴리오"라 그 명칭은 중복이라는 논의 끝에 "프로젝트"로 결정
   - 전체 9개 HTML(`index.html`, `resume/`, `career/` 하위 5개, `ai-tool/`, `prototypes/`)의 내비게이션 라벨을 일괄 변경, 홈 화면 카드 아이콘도 🤖 → 🚀로 교체

10. **DEMO 섹션 레이아웃 통일**
    - DEMO 01/02가 `.tool-breakout`으로 본문 컨테이너(880px)를 벗어나 최대 1240px까지 넓어져, 타임라인·카드와 폭이 어긋나던 문제 수정
    - 래퍼 제거 후 나머지 섹션과 동일한 컨테이너 폭에 정렬, 미사용 CSS 규칙(`.tool-breakout`) 삭제
    - `.tool-desc`에 걸려있던 `max-width: 72ch`도 제거 — 타임라인 문단(`.project-desc`)과 달리 데모 설명 텍스트만 좁게 잡혀 우측 여백이 크게 남던 불일치 해소

11. **한글 줄바꿈 및 DataViewer 에러 표시 개선**
    - `body`에 `word-break: keep-all` 추가 — 기본 CJK 줄바꿈 규칙 때문에 "상자·아이템"처럼 단어 중간에서 끊기던 문제를 사이트 전역에서 해결
    - DataViewer 데모 아래에 안내 박스 추가: "String 없음"/"0건 (미사용)" 주황색 표시가 끊긴 참조·미사용 데이터를 찾기 위한 표시라는 설명 삽입
    - 표 우측 하단 "N / N개 표시" 옆에 `Error N건` 배지 추가 — 현재 탭에서 끊긴 참조(Item)·미사용 건수(BaseItem)를 실시간 집계 (`ai-tool.js`)
