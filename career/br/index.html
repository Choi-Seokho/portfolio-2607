2014.09 ~ 2017.11 · 기획팀 사원 · Mobile RTS Tales Craft

조작 방식 정의

모바일 RTS Tales Craft의 터치 조작 체계를 레퍼런스 없이 플레이 테스트를 반복하며 직접 정립했습니다. 터치 인터페이스라는 제약 안에서 스타크래프트 같은 PC RTS에서 쓰이는 조작을 거의 대부분 구현해, 유닛 선택·이동부터 건설·스킬·업그레이드까지 하나의 일관된 규칙으로 묶었습니다. 테스트를 거듭하며 굳힌 조작 규칙들은 최종적으로 조작 가이드 문서로 정리해 개발팀·QA의 공통 기준으로 삼았습니다.

목적

버튼이 많아 조작이 복잡해지고, 터치 미스 등 오류가 발생하는 문제를 완화

해결 방법

구분문제점해결 방법
유닛 선택 그룹 선택 버튼을 누른 뒤 사각형으로 드래그하는 방식(스타크래프트 방식)은 조작 단계가 많아 번거로웠음 경로를 따라 아군 유닛을 선택하는 한 붓 그리기 방식으로 변경. 단일 선택(터치)·다중 선택(경로 드래그)으로 규칙 확정
이동/공격 이동 전환 이동/공격 이동 전환이 버튼 토글 방식이라 빠르게 전환하기 어려웠음 1회 터치는 이동, 2회 이상 터치는 공격 이동(어택땅)으로 전환(연타하는 플레이 패턴 반영). 이동/공격 이동 동작을 서로 변경할 수 있는 옵션 추가
취소 동작 기능마다 취소 동작 방식이 달라 헷갈리기 쉬웠음 모든 취소 동작을 아이콘을 위/아래로 플릭하는 동작 하나로 통일
건물 건설 / 스킬 사용 초기에는 클릭 투 클릭 방식이라 클릭 미스가 발생하기 쉬웠고, 정교한 위치 지정이 어려웠음 Drag & Drop으로 변경, 드래그 중 더미(예상 위치) 표시로 정교한 배치 가능. 아이콘 자리로 되돌리면 취소되는 규칙을 건설·스킬에 동일 적용

위와 같은 방식으로 출시까지 조작 방식을 체계화함

인게임 진입 시 첫 화면: 자원 현황·일꾼·병력 생산·미니맵 등 UI 요소 정의

유닛 근처에서 Drag & Drop 시 이동 경로 상의 유닛을 다중 선택하는 조작 정의

유닛 선택 해제: 하단에 표시된 유닛 아이콘을 위로 밀면 해당 유닛이 선택 해제됨

스킬 사용: 스킬 아이콘을 원하는 영역으로 Drag & Drop해 시전, 건설과 동일하게 아이콘 자리로 되돌리면 취소되도록 규칙 통일

레벨 디자인

PvE 반복 콘텐츠 ‘천공의 섬(Infinity Battle)’의 스테이지 진입 구조와 전투 방식을 기획하고, 하나의 공용 MAP으로 여러 Stage를 운용할 수 있도록 적 배치 지점·공격 방향·아군 시작점을 표준화된 레이아웃으로 설계했습니다.

진입 조건진입 시 타이머 노출, 영웅 최대 3명 소환(자원·인구 제한 없음), 모든 적 처치 시 승리
MAP 구성공용 Map에 적 배치 지점 3곳 + 아군 시작점 1곳을 두어 스테이지별 난이도만 다르게 구성
진행 구조화면보다 큰 월드맵을 좌우 스크롤하며 Stage를 순차적으로 오픈하는 진행 구조 설계

신규 콘텐츠를 제안할 때는 이처럼 스테이지 진입 구조·전투 방식·MAP 구성을 기획 문서로 정리해 제안서를 작성합니다.

천공의 섬 기획 문서 8p: 전투 방식(영웅 소환)과 MAP 구성(적군 배치 지점·아군 시작점·공격 방향)을 함께 정리

맵 제작 예시 (PvP)

PvP 밸런스를 고려해 직접 제작한 맵 중 일부입니다. 이미 존재하는 콘텐츠는 빠르게 테스트 가능한 레벨을 만들어 플레이 테스트를 진행한 뒤 결과를 정리하는 방식으로 작업합니다.

공격대 시스템

생산 단계 없이 Preset으로 지정한 유닛(영웅+병력)을 즉시 투입하는 ‘공격대(Attacking Forces)’ 전투 시스템을 기획했습니다. 자원 채집 → 건설 → 생산 → 전투로 이어지는 기존 RTS 전투와 달리, 이벤트 전투·거점전·보스전처럼 채집·건설·생산 단계를 생략한 콘텐츠에 사용되는 별도 전투 체계입니다.

목적

PvP 외에도 반복 플레이가 가능한 콘텐츠가 필요했음

해결 방법

모든 유닛을 자유롭게 쓸 수 있으면 가장 강한 유닛으로만 클리어하는 단순 반복 플레이가 되기 쉬우므로, 보유한 유닛 안에서 덱을 구성해 플레이하는 방식으로 콘텐츠를 설계

UI 구성영웅 슬롯 + 병력 슬롯(연방 상극인 라미아/타이탄 동시 배치 불가, 총 12슬롯)으로 구성된 Attacking Forces UI 설계
배치 방식영웅은 Drag & Drop, 병력은 터치·홀드(COC류 게임의 병력 생산 참고)로 배치·해제하는 입력 방식 정의
업그레이드공격대 업그레이드로 영웅 슬롯 Unlock 및 배치 가능 병력 인구수 증가
전투력 수치화유닛별 초당 공격력(공격력 ÷ 공격 속도)을 합산한 Forces ATK 수치로 공격대 전투력을 수치화

공격대 UI 전체 배치: 영웅/병력 배치 슬롯, Forces ATK, 소대 전환, 업그레이드 버튼 정의

영웅 배치 슬롯: Drag & Drop을 통한 배치·해제, 배치 시 아이콘 상태 변화 정의

공격대 업그레이드: 영웅 슬롯 Unlock, 병력 인구수 증가 등 업그레이드 요소 설계
