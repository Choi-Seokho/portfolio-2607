# 포트폴리오 사이트 작업 이력

최종 갱신: 2026-07-14

## 배경

게임 기획자(시스템/전투/레벨) 이직용 포트폴리오 정적 사이트 제작. `포트폴리오_제작_계획안.md`를 기반으로 이력서 정리 → 사이트 레이아웃 구축 → AI 활용/프로토타입 콘텐츠 채우기 → 네비게이션 정리 순으로 진행.

## 진행 순서

1. **이력서 전체 정리** (`이력서/이력서_전체.md`)
   - 게임잡 등록 이력서 PDF/PNG 5장을 참고해 경력 4곳(OneWayTicketStudio·Factorial Games·FIT·BR Games), 보유역량, 교육이수, 자기소개서 5키워드, 병역사항까지 마크다운으로 전체 정리
   - 경력별 연봉 정보 등 누락분 보완, 사용자 확인 후 확정 (증명사진·플랫폼 확인 문구·인쇄 메타데이터는 제외하기로 결정)

2. **사이트 기본 레이아웃 구축**
   - 정적 다중 페이지 구조: `index.html` + `resume/`, `career/`(회사별 하위 폴더), `ai-tool/`, `prototypes/`, `assets/`
   - 공통 스타일시트(`assets/css/style.css`) — 라이트/다크 테마 자동 대응
   - `robots.txt` + 각 페이지 `noindex` 메타로 검색엔진 차단
   - 이력서 페이지에 전체 콘텐츠(경력 타임라인, 자기소개서 등) 반영
   - 페이지 전환 시 네비게이션 위치가 흔들리던 문제 → `html { overflow-y: scroll }`로 스크롤바 공간을 항상 확보해 해결
   - 경력기술서 목록에서 누락됐던 FIT 카드 추가
   - 소개 페이지·이력서 페이지에 `tel:`/`mailto:` 클릭 가능한 연락처 링크 추가

3. **프로토타입 콘텐츠 추가 (제작대 리뉴얼 / 일일 미션)**
   - `클루드/.../design/` 폴더의 실제 React 프로토타입 HTML(craft-station, daily-mission)을 `prototypes/`에 그대로 게시 (데모 앱, 사이트 네비게이션 없이 독립 실행)
   - 사내 기획서 PDF(`작성 문서/OWT/`)의 한글이 깨지는 문제 발견 → 원인은 시스템 `pdftotext`가 실제로는 2017년산 구버전 Xpdf였기 때문. `pdfjs-dist` 기반 추출 도구를 `C:\Users\sinji\.claude\tools\pdf-extract\`에 영구 설치해 해결 (세션에 종속되지 않음, 메모리에도 기록해둠)
   - PDF/설계 분석 md 문서 내용을 바탕으로 기획 요약(핵심 규칙, 주요 기능, UI/데이터 구조, 사운드 디자인, 설계 반복 하이라이트) 작성
   - 민감 정보(실제 아이템 ID·확률 수치, 사내 Confluence/Slack 링크, 동료 실명, 치트 명령어 등)는 의도적으로 제외

4. **네비게이션 구조 정리**
   - 상단의 "프로토타입" 탭을 없애고 "AI 활용" 하위 서브탭으로 통합: `🤖 AI 툴 활용 | 🎮 기획 및 프로토타입`
   - 프로토타입 페이지를 카드 2개(제작대/일일미션, 각각 별도 기획서 페이지) → **한 페이지 목록형**으로 재구성 (상단 점프 링크 + 두 섹션을 구분선으로 나열)
   - 개별 기획서 페이지(`craft-station/design.html`, `daily-mission/design.html`)는 내용을 목록 페이지에 병합한 뒤 삭제
   - 메인 소개 페이지의 "AI 활용 사례" / "프로토타입" 카드도 "AI 활용" 카드 하나로 통합

5. **맨 위로 이동 버튼**
   - 전 페이지 공통으로 우측 하단 고정 버튼 추가 (`assets/js/back-to-top.js` + `style.css`의 `.back-to-top`)
   - 400px 이상 스크롤 시 서서히 나타나고, 클릭 시 부드럽게 최상단으로 스크롤

6. **배포 방식 논의**
   - GitHub Pages 무료 배포는 리포가 반드시 **public**이어야 함 → `robots.txt`/`noindex`는 검색엔진 노출만 막을 뿐, git 커밋 히스토리·GitHub 자체 검색으로는 노출될 수 있음
   - 대안(Netlify Drop, private repo + Netlify/Vercel)도 검토했으나, 2026-07-15 재개 시 **단순함 우선**으로 public repo + GitHub Pages로 최종 결정
   - MapGenie 맵 이미지(LV_BCF_A/B) 실사용 건은 OneWayTicketStudio 대표 컨펌 완료. career/ 페이지는 전부 "준비중" placeholder로 실제 민감 콘텐츠 없음을 확인
