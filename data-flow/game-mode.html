🔗 데이터 흐름 · 게임 모드 → 페이즈 프로세스

게임 모드가 시작하는 지점부터, 시간에 따라 이어지는 페이즈 체인까지의 흐름을 dbdiagram.io로 그려봤습니다.

이 주제는 사전에 정리된 기획 문서가 없어, 실제 서비스 중인 DT_GameMode / DT_GameMode_Group / DT_PhaseProcess 원본 데이터를 직접 분석해서 구성했습니다. QA 전용 플래그 등 핵심 흐름과 무관한 필드는 생략했고, 실 밸런스 수치·사내 문서 링크도 포함하지 않았습니다. 아래는 dbdiagram.io의 "DBML-in-Link" 뷰어를 임베드한 것입니다.

개요

게임 모드 하나는 StartPhaseProcess_ID 딱 하나만 갖고 있고, 실제 라운드 진행은 DT_PhaseProcess가 NextPhase_ID로 자기 자신을 참조하며 만드는 체인을 따라 흘러갑니다. 각 페이즈마다 폐쇄되는 층 수, 독 피해, 새로 열리는 탈출구 수가 데이터로 정의되어 있어, 시간이 지날수록 위협은 올라가고 탈출 기회는 늘어나는 곡선을 코드 수정 없이 데이터만으로 튜닝할 수 있습니다.

데이터 관계도

↗ 새 탭에서 크게 보기 (임베드가 보이지 않으면 이 링크를 이용해주세요)

설계 포인트

  1. 페이즈 체인(연결 리스트) 구조: NextPhase_ID로 자기 자신을 참조해, 100개가 넘는 시간 단계가 하나의 체인으로 이어집니다. 게임 모드는 시작점 하나만 가리키면 되고, 이후 진행은 체인을 타고 흐릅니다.
  2. 점진적 위협 상승: 페이즈마다 층 폐쇄 수·독 피해·탈출구 개수가 데이터로 조절되어, 밸런스 변경 시 코드가 아니라 페이즈 데이터만 수정하면 됩니다.
  3. 모드 그룹과 개별 모드 분리: 상위 그룹(예: Rookie, Normal, Hardcore)과 인원 구성별 개별 모드(솔로, 듀오, 트리오)를 나눠, 같은 그룹 안에서도 인원 구성별로 다른 시작 페이즈를 지정할 수 있게 설계했습니다.
  4. 다른 시스템과의 연결점: 일일 미션의 진행 가능 모드 조건이 여기 DT_GameMode_Group의 그룹 이름을 그대로 참조합니다.

본 페이지는 채용 지원 목적의 비공개 포트폴리오입니다.
