# 게임 기획자 포트폴리오 사이트 제작 계획안

## 목표
게임 기획자(시스템/전투/레벨) 이직용 포트폴리오를 HTML 기반 정적 웹사이트로 제작한다.
index.html(또는 배포된 사이트 링크) 하나로 이력서, 경력기술서, AI 활용 사례, 프로토타입까지 모두 열람 가능해야 한다.

## 사이트 구조 (제안)

```
portfolio/
  index.html          # 메인 진입점 (전체 네비게이션 + 소개)
  resume/              # 이력서 + 자기소개서
  career/               # 경력기술서 (회사별)
    factorial/
    oneway/
    br/
  ai-tool/              # AI 활용 기획 툴 (정적 데모 버전)
  prototypes/           # 기획서 작성 시 만든 프로토타입
  assets/                # 공통 css/js/이미지
```

## 구현 방식

- **정적 다중 페이지 + 공통 헤더/네비게이션** 방식으로 진행 (SPA 아님)
  - 이유: 유지보수 쉽고, 구현 난이도 낮음. 포트폴리오 규모(페이지 5~8개 내외)에 SPA는 과함
- 각 페이지는 상대경로로 연결되어, 로컬 실행이든 배포 후 접속이든 동일하게 작동해야 함
- 회사별 경력기술서는 단순 텍스트 나열보다 타임라인/카드 형태의 시각화 권장

## 포함될 콘텐츠

1. **이력서 (자소서 포함)**
   - 기존에 다듬어둔 자소서 텍스트 활용 (별도 파일 있음)
2. **경력기술서**
   - 회사별(Factorial Games / OneWayTicketStudio / BR Games) 문서 기반 업무 설명
3. **AI 활용 내용**
   - 기존에 제작했던 Game Design 툴을 서버 없이, 미리 입력된 가데이터로 동작하는 정적 버전으로 이관
   - ⚠️ 전체 기능 재현 X — 가장 인상적인 기능 1~2개만 선정해 깊이 있게 데모할 것
   - 상세 계획은 아래 "AI 활용 페이지 상세 계획" 섹션 참고
4. **프로토타입**
   - 기획서 작성 시 만들었던 프로토타입 첨부
   - 형태(영상/웹/실행파일)에 따라 첨부 방식 결정 필요 (아래 미결정 항목 참고)

## 호스팅 & 배포

- **GitHub Pages**로 배포 (무료, `https://아이디.github.io/저장소이름/` 형태의 링크 생성)
- 검색엔진 노출 방지:
  - `robots.txt`로 크롤링 차단
  - 각 페이지에 `<meta name="robots" content="noindex">` 추가
- 지원서에는 링크를 텍스트로만 기재 (파일 다운로드/압축 방식 대신 채택)
  - 이유: 로컬 zip 첨부는 지원 폼의 첨부 형식 제약, 보안 정책상 파일 실행 차단, 로컬 실행 시 리소스 경로 깨짐 등의 리스크가 있음

## AI 활용 페이지 상세 계획

### 톤
경력기술서에서 기획 역량은 이미 다루므로, 이 페이지는 **"Claude Code를 어떻게 활용해서 무엇을 만들고 개선했는가"** 자체를 주인공으로 삼는다. AI 활용 역량 중심, 기획 서사는 곁들이는 정도.

### 페이지 섹션 구조
1. **Hero**: 임팩트 한 줄 + 정량 지표(모듈 수, 자체 MCP 서버 수, 세션 로그 수 등)
2. **활용 프로세스 타임라인**: 세션 로그 체계화 → MCP 서버 구축 → 반복 작업의 스킬화 → MapGenie 반복 개선 과정을 카드형으로
3. **핵심 데모 ① MapGenie**: 스폰 위치 시각화 도구, 타입 필터/검색 인터랙션 + "이렇게 반복 개선했다" 미니 타임라인(검색 버그 수정 → 캐싱 → 타입별 필터 오버레이 → SVG 아이콘 → Enum 한글 연동 → UI 간소화)
4. **핵심 데모 ② DataViewer**: SQL 유사 쿼리로 가상 게임 데이터 테이블 조회하는 간단 데모. "최초 모듈이자 다른 도구들의 기반이 된 인프라"라는 연결고리로 소개
5. **AI 협업 프로세스 상세**: 자체 구축 MCP 서버 2개(`tmw-analyzer`, `confluence-mcp-server`), 반복 작업을 전용 스킬로 고정한 사례(`/new-log-viewer` 등), `session-insights` 메타 자기개선 루프 — 스크린샷/코드 스니펫 위주 서술형
6. **원칙 고지 문구**: 데이터/구조는 재구성되었다는 점 명시

### 데모별 데이터 처리 방침
- **MapGenie**: 배경 맵 이미지는 **실사용 확정** (대표 컨펌 완료). 단, 스폰 위치/수량 등 구체 데이터는 가상 데이터로 대체
- **DataViewer**: 테이블명·컬럼명·데이터 전부 **가칭/가상으로 재구성** (예: `DT_Item` → `DT_SampleItem` 류의 이름, 실제 스키마 노출 금지)

### 작업 순서
1. 이 계획 확정 (완료)
2. MapGenie 데모 구현 (실제 맵 이미지 + 가상 스폰 데이터 + 필터/검색 인터랙션 + 반복개선 타임라인)
3. DataViewer 데모 구현 (가칭 테이블 + 쿼리 UI)
4. AI 협업 프로세스 서술 섹션 작성
5. `ai-tool/index.html` placeholder를 실제 콘텐츠로 교체
